# クリダス Webアプリ受け入れテスト

## 1. 目的

このドキュメントは、クリダス Webアプリ全体を本番公開前に確認するための受け入れテスト項目一覧です。

機能単位ではなく、実際の利用者の行動や業務フローに沿って確認できるように整理しています。

---

## 2. 対象範囲

今回の受け入れテストでは、以下を対象にします。

- 認証/ログイン導線
- ベンダー向け管理画面
- 主催者向け管理画面
- 公開ページ/外部導線
- モバイルオーダー
- LINE通知
- Stripe決済
- 商品/出店/応募/集計の主要導線
- 権限制御
- 例外系

---

## 3. テスト実施前の前提

### 環境

- ステージング環境: `https://stg.vnd.kuridas-os.jp`
- LINE LIFF
- Stripe Sandbox
- Supabase 接続済み

### 用意しておくテストアカウント

- ベンダーアカウント 1件以上
- 主催者アカウント 1件以上
- LINE通知確認用の一般ユーザー 1件以上

### 用意しておくテストデータ

- 出店場所
- 商品
- オプション
- 営業枠
- QRコード/固定注文URL
- Stripe テストカード

### Stripe テストカード

- カード番号: `4242 4242 4242 4242`
- 有効期限: 未来日
- CVC: 任意の3桁

---

## 4. テスト観点一覧

### 4-1. 共通/全体

- ページ遷移時に原則ページ最上部へスクロールされる
- PCで主要画面が崩れず表示される
- スマホで主要画面が崩れず表示される
- ログイン状態に応じて適切な画面へ遷移する
- ログアウト後に保護ページへ直接入れない
- ブラウザ再読み込み後も破綻しない
- エラーメッセージが利用者に理解できる文言になっている

### 4-2. 認証/アカウント

- ベンダー新規登録が完了する
- 主催者新規登録が完了する
- メール確認後に想定画面へ遷移する
- ログインが成功する
- ログアウトが成功する
- ベンダーが主催者画面へ不正遷移できない
- 主催者がベンダー画面へ不正遷移できない
- ロール選択画面が正しく動く

### 4-3. ベンダープロフィール/初期設定

- ベンダープロフィールを登録できる
- 必須項目未入力時に保存できない
- ロゴ画像や説明文が保存される
- 保存後、再表示して内容が保持される

### 4-4. 主催者プロフィール/初期設定

- 主催者プロフィールを登録できる
- 必須項目未入力時に保存できない
- 保存後、再表示して内容が保持される

### 4-5. 出店場所管理

- 出店場所を新規作成できる
- 出店場所を編集できる
- 一覧に反映される
- 削除可否や非表示制御が想定どおり動く

### 4-6. 募集/オファー管理

- 主催者が募集を作成できる
- 募集一覧に表示される
- ベンダーが募集一覧を閲覧できる
- ベンダーが応募できる
- 応募後に応募状況へ反映される
- 主催者が応募内容を確認できる
- 主催者/ベンダー双方でステータスが整合する

### 4-7. 商品マスタ/商品管理

- 商品を新規作成できる
- 商品名、価格、説明、画像が保存される
- 商品を編集できる
- 公開/非公開を切り替えられる
- 売り切れ表示を切り替えられる
- 表示順が反映される

### 4-8. オプション管理

- オプショングループを作成できる
- 単一選択/複数選択を設定できる
- 必須/任意を設定できる
- 選択肢を追加できる
- 価格加算が反映される
- 商品への紐付けができる

### 4-9. モバイルオーダー 営業枠管理

- 営業枠を新規作成できる
- 営業開始/終了時刻が保存される
- 営業中のみ注文受付になる
- 営業時間外は受付停止表示になる
- 次回営業枠が正しく表示される

### 4-10. モバイルオーダー 在庫管理

- 初期在庫を営業枠ごとに設定できる
- 在庫調整を加算/減算で記録できる
- 調整履歴が表示される
- 残数が正しく計算される
- 売り切れ時に注文不可になる
- 残りわずか表示が正しく出る
- 在庫未設定時に在庫準備中表示になる

### 4-11. モバイルオーダー 公開注文導線

- 固定注文URLで注文ページを開ける
- LIFF URLから注文ページを開ける
- ストア名、営業時間、商品一覧が表示される
- 商品選択後、選択中の状態が分かる
- オプション選択ができる
- 数量変更ができる
- カート追加ができる
- カートページへ遷移できる
- 注文内容確認ページへ遷移できる
- 支払い前確認モーダルが表示される

### 4-12. モバイルオーダー 注文/決済

- 注文番号が正しい形式で採番される
- 同日連番が想定どおり増える
- 翌日営業枠で連番がリセットされる
- Stripe Checkoutへ遷移できる
- テストカードで決済できる
- 決済成功後に注文完了画面へ戻る
- 注文完了画面で注文番号、ニックネーム、金額が表示される
- `もう一度注文ページを見る` で通常注文画面へ戻れる

### 4-13. モバイルオーダー LINE通知

- 注文完了通知が自動送信される
- 調理開始通知が自動送信される
- 完成通知が自動送信される
- 通知文面に注文番号が入る
- 通知文面に注文商品と個数が入る
- 通知履歴に送信結果が残る
- 失敗時に再送できる

### 4-14. モバイルオーダー ベンダー注文管理

- 注文ダッシュボードで新規注文を確認できる
- `受付済` の件数が正しく集計される
- `調理中` の件数が正しく集計される
- `完成` の件数が正しく集計される
- `受取済` の件数が正しく集計される
- 注文詳細に商品内容、オプション、通知状況が表示される
- `調理開始` に更新できる
- `完成` に更新できる
- `受取済` に更新できる

### 4-15. QRコード/公開導線

- モバイルオーダー設定トップでQRが表示される
- URLコピーができる
- QRから注文ページを開ける
- LIFF導線と通常URL導線の両方が成立する

### 4-16. CSV/アップロード系

- CSVアップロード画面が開く
- 想定フォーマットのCSVを取り込める
- 不正フォーマットのCSVでエラー表示される
- 取り込み結果が一覧へ反映される

### 4-17. 分析/集計

- クロス分析画面が開く
- 日別集計画面が開く
- 分析条件変更で表示が更新される
- データ0件時でも画面が破綻しない

### 4-18. 例外系

- 営業時間外に注文ページを開くと受付停止表示になる
- 売り切れ商品を注文できない
- 在庫不足の商品を注文できない
- 必須オプション未選択でカート追加できない
- 決済キャンセル後に注文完了扱いにならない
- Webhook未着時に決済確認エラーが利用者に分かる
- 不正な token の注文URLで注文できない
- ログイン不要ページからログイン必須画面へ不正遷移しない

## 5. 業務フロー別テストシナリオ

### シナリオA: ベンダー初期設定から注文受付開始まで

1. ベンダー登録/ログイン
2. ベンダープロフィール登録
3. 商品作成
4. オプション設定
5. 営業枠作成
6. 初期在庫設定
7. QR/URL確認

期待結果

- すべて保存できる
- 注文ページに商品が公開される
- 営業時間内のみ受付中になる

### シナリオB: 一般ユーザーが注文して受け取り完了まで

1. LIFFまたはQRから注文ページを開く
2. 商品選択
3. カート確認
4. 注文内容確認
5. 支払い前確認モーダル確認
6. Stripe決済
7. 注文完了通知受信
8. ベンダーが調理開始
9. 調理開始通知受信
10. ベンダーが完成
11. 完成通知受信
12. 店頭受け渡し
13. ベンダーが受取済に更新

期待結果

- 注文番号が採番される
- ベンダー側ダッシュボードに注文が見える
- 3段階のLINE通知が届く
- 最終的に受取済まで完了する

### シナリオC: 在庫不足と売り切れ対応

1. 在庫管理ONの商品を用意する
2. 初期在庫を少量で設定する
3. 注文を繰り返して残数を減らす
4. 残りわずか表示を確認する
5. 売り切れ時に注文不可を確認する
6. ベンダー側で在庫調整を行う

期待結果

- 残数表示が変化する
- 売り切れ時は注文できない
- 在庫調整後に表示が追従する

### シナリオD: 決済キャンセル/失敗

1. 注文確認まで進む
2. Stripe画面でキャンセルする
3. 注文画面へ戻る
4. 再度決済へ進む

期待結果

- 未決済注文が完了扱いにならない
- 利用者に分かるエラーまたは案内が出る
- 再度決済に進める

### シナリオE: 権限制御

1. ベンダーで主催者画面URLを直接開く
2. 主催者でベンダー画面URLを直接開く
3. 未ログイン状態で管理画面URLを開く

期待結果

- 不正アクセスできない
- 適切な画面へ戻される

## 6. 受け入れ判定の目安

### 合格

- 主要業務フローが1周通る
- 決済/通知/注文管理が成立する
- 権限不整合がない
- 利用者にとって致命的なUI崩れがない

### 要修正

- 注文できない
- 決済できない
- 通知が届かない
- 在庫や注文数が不整合
- 役割ごとの画面制御が崩れる
- 公開導線でログイン要求や不正遷移が起こる
