# モバイルオーダー画面ワイヤー仕様

## 1. 方針

MVPでは、画面数を増やしすぎず、現場で迷わないことを優先します。

- ベンダー側は `すぐ操作できる`
- ユーザー側は `迷わず注文できる`
- 店頭運用では `注文番号とニックネームで受け渡せる`

## 2. ベンダー画面ワイヤー

### 2-1. モバイルオーダー設定トップ

目的

- ベンダーが今日の営業状態と注文対応状況をすぐ確認する

主な要素

- ページタイトル: `モバイルオーダー`
- 店舗セレクター
- 今日の受付状態カード
- 次回営業枠カード
- 未対応注文数カード
- 本日注文件数カード
- 固定QRコードカード
- 主要導線ボタン

ワイヤー

```text
+--------------------------------------------------+
| モバイルオーダー                      [店舗切替] |
+--------------------------------------------------+
| [受付中] 今日の受付 11:00-15:00                 |
| [次回営業] 4/28 11:00-15:00                     |
| [未対応 5件] [本日注文 18件]                    |
+--------------------------------------------------+
| 固定QRコード                                    |
| [ QR ]  https://.../order/abc123               |
| [印刷用表示] [ダウンロード]                     |
+--------------------------------------------------+
| [商品管理] [オプション管理] [営業スケジュール]   |
| [注文ダッシュボード]                            |
+--------------------------------------------------+
```

### 2-2. 商品一覧画面

目的

- 商品の公開状態と売り切れ状態をすばやく管理する

主な要素

- 新規商品追加ボタン
- 商品カードまたは表
- 商品画像
- 商品名
- 価格
- 公開状態トグル
- 売り切れトグル
- 編集ボタン

ワイヤー

```text
+--------------------------------------------------+
| 商品管理                               [新規追加] |
+--------------------------------------------------+
| [画像] チキンオーバーライス  ¥900               |
| 公開中 / 販売中                     [編集] [売切]|
+--------------------------------------------------+
| [画像] タコライス                ¥850            |
| 非公開 / 販売中                   [編集] [公開]  |
+--------------------------------------------------+
```

### 2-3. 商品編集画面

目的

- 商品情報を登録、更新する

入力要素

- 商品名
- 商品説明
- 価格
- 商品画像
- 表示順
- 公開状態
- 売り切れ状態
- 適用オプショングループ

ワイヤー

```text
+--------------------------------------------------+
| 商品編集                                         |
+--------------------------------------------------+
| 商品名 [____________________________]            |
| 説明   [____________________________]            |
| 価格   [________] 円                             |
| 画像   [アップロード]                            |
| 表示順 [__]                                      |
| [ ] 公開する   [ ] 売り切れ                      |
| 適用オプション                                   |
| [x] トッピング  [x] 辛さ  [ ] ドリンクセット     |
|                                      [保存]      |
+--------------------------------------------------+
```

### 2-4. オプション管理画面

目的

- 商品ごとの選択肢ルールを作る

主な要素

- オプショングループ一覧
- 必須/任意
- 単一/複数選択
- 選択肢と加算額
- 編集導線

ワイヤー

```text
+--------------------------------------------------+
| オプション管理                         [新規追加] |
+--------------------------------------------------+
| トッピング  任意 / 複数選択                     |
| 目玉焼き +100 / チーズ +100 / アボカド +150      |
|                                          [編集]  |
+--------------------------------------------------+
| 辛さ        必須 / 単一選択                      |
| 普通 +0 / 中辛 +0 / 辛口 +0                      |
|                                          [編集]  |
+--------------------------------------------------+
```

### 2-5. 営業スケジュール管理画面

目的

- 固定QRに対して、注文受付可能な営業日時を管理する

主な要素

- カレンダーまたは日付リスト
- 営業日
- 開始時刻
- 終了時刻
- ステータス
- 当日受付停止ボタン

ワイヤー

```text
+--------------------------------------------------+
| 営業スケジュール                      [営業枠追加] |
+--------------------------------------------------+
| 4/27(月) 11:00-15:00  受付中           [停止]    |
| 4/28(火) 11:00-15:00  予定             [編集]    |
| 4/29(水) 休業                         [編集]    |
+--------------------------------------------------+
```

### 2-6. 注文ダッシュボード

目的

- 注文対応を1画面で処理する

主な要素

- フィルタ: 未対応 / 調理中 / 完成
- 注文カード
- 注文番号
- 注文時刻
- ニックネーム
- 商品明細
- ステータス更新ボタン

ワイヤー

```text
+--------------------------------------------------+
| 注文ダッシュボード                               |
| [未対応] [調理中] [完成] [全件]                  |
+--------------------------------------------------+
| A0001  11:08  たろう                             |
| チキンオーバーライス x1 / 目玉焼き               |
| [調理中にする] [詳細]                            |
+--------------------------------------------------+
| A0002  11:10  ゆき                               |
| タコライス x2                                    |
| [完成にする] [詳細]                              |
+--------------------------------------------------+
```

## 3. ユーザー画面ワイヤー

- 注文導線の入口

主な要素

## 4. 画面ごとの優先順位

### MVPで必須

- モバイルオーダー設定トップ
- 商品一覧
- 商品編集
- オプション管理
- 営業スケジュール管理
- 注文ダッシュボード
- QRコード発行
- 店舗注文トップ
- 商品詳細モーダル
- カート確認
- 決済画面
- 注文完了画面
- 受付停止画面

### MVPでできれば欲しい

- 注文状況確認画面
- 注文詳細モーダル

## 5. UI実装メモ

- ベンダー画面は既存 `AppShell` と `HeaderNav` に載せる
- ベンダー一覧系は既存の `soft-panel` 系スタイルを流用できる
- ユーザー側は LINE 内ブラウザ前提で `片手操作しやすい固定CTA` を優先
- 注文番号は完了画面で最も目立つ要素にする
- 受付停止画面でも `次回受付予定` は必ず見せる
